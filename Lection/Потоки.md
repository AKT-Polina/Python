# Управление внешними процессами
Модуль subprocess появился в Python 2.4 как универсальное решение для запуска внешних процессов
## Функция subprocess.run() 
Функция subprocess.run() — это центральный компонент модуля, предназначенный для простого и эффективного запуска команд. Введённая в Python 3.5, она стала рекомендуемым способом вызова внешних процессов благодаря своей выразительности и интуитивно понятному API.

Базовый синтаксис выглядит так:

```Python
subprocess.run(args, *, stdin=None, input=None, stdout=None, stderr=None, 
capture_output=False, shell=False, cwd=None, timeout=None, 
check=False, encoding=None, errors=None, text=None, env=None, 
universal_newlines=None)
```

Функция возвращает объект CompletedProcess, содержащий информацию о выполненном процессе, включая статус завершения и вывод.

Рассмотрим основные параметры функции run():

- args: Команда для выполнения. Может быть строкой (при shell=True) или списком аргументов
- capture_output: Если True, перенаправляет stdout и stderr в память
- check: Если True, вызывает исключение при ненулевом коде возврата
- shell: Использовать ли оболочку системы для выполнения команды
- timeout: Максимальное время выполнения в секундах
- text: Если True, преобразует вывод из байтов в строки
Давайте посмотрим на практические примеры использования run():

```python
# блокнота (если это не стандартная программа Windows, нужно будет указать путь)
subprocess.run(['notepad.exe'])

#запуск блокнота с параметром
subprocess.run(['notepad.exe', "C:\\Users\\304.PC-KLASS.005\\Desktop\\1.txt"])

#Запуск консольной команды
subprocess.run(['echo', "Hello world!"])

# Установка тайм-аута (пример под Linux)
try:
result = subprocess.run(['sleep', '10'], timeout=5)
except subprocess.TimeoutExpired:
print("Process timed out")

```

```python
import subprocess

# Запускаем 'ping' (в Windows ping -n 1 google.com)
res = subprocess.run(["ping", "-c", "1", "8.8.8.8"], capture_output=True, text=True)

if res.returncode == 0:
    print("Вывод команды:\n", res.stdout)
else:
    print("Ошибка:\n", res.stderr)

```

## Класс Popen предоставляет несколько важных методов для управления процессами:

- communicate(): Отправляет данные в stdin и читает из stdout/stderr до завершения процесса
- poll(): Проверяет, завершился ли процесс (возвращает код возврата или None)
- wait(): Блокирует выполнение до завершения процесса
- terminate(): Прерывает процесс сигналом SIGTERM
- kill(): Принудительно останавливает процесс сигналом SIGKILL

```python
import subprocess
import time

# Запускаем процесс 
proc = subprocess.Popen(["notepad.exe"]) # Для Linux: ["sleep", "60"]

# 1. Проверить, работает ли процесс прямо сейчас
if proc.poll() is None:
    print("Процесс еще запущен")
else:
    print(f"Процесс завершен с кодом {proc.returncode}")

# 2. Узнать системный ID (PID)
print(f"ID процесса в системе: {proc.pid}")

```

# Завершение процесса
Есть нормальный способ через os:
```python
import os

pid = 1234
os.kill(pid, 9) # Аналог Task Kill

```
А есть возможность запускать завершать через уже знакомый нам subprocess простой виндсовской/линуксовской командой
```python
import subprocess

pid = 1234

# В Windows:
subprocess.run(["taskkill", "/F", "/PID", str(pid)])

# В Linux/macOS:
subprocess.run(["kill", "-9", str(pid)])

```

# Просмотр процессов
Для просмотра запущенных процессов:
```python
import subprocess
import platform

def get_process_list():
    # Определяем команду в зависимости от ОС
    if platform.system() == "Windows":
        cmd = ["tasklist"]
    else:
        # Для Linux и macOS
        cmd = ["ps", "aux"]

    try:
        # Запускаем команду и захватываем вывод
        result = subprocess.run(cmd, capture_output=True, text=True, check=True)
        return result.stdout
    except subprocess.CalledProcessError as e:
        return f"Ошибка при выполнении команды: {e}"

print(get_process_list())

```


# Управление процессами
Синтаксис почти идентичен threading, но импортируется другой класс.
```python
import multiprocessing
import time

def compute_heavy_task(name):
    print(f"Процесс {name} начал вычисления...")
    result = sum(i * i for i in range(10**7)) # Тяжелая задача
    print(f"Процесс {name} закончил: {result}")

if __name__ == "__main__":
    # Создание процесса
    p1 = multiprocessing.Process(target=compute_heavy_task, args=("A",))
    p2 = multiprocessing.Process(target=compute_heavy_task, args=("B",))

    p1.start()
    p2.start()

    p1.join()
    p2.join()

```
 В Windows код создания процессов обязательно должен находиться внутри блока if __name__ == "__main__":, иначе программа будет бесконечно пытаться запустить саму себя 

 Для массовых задач удобнее использовать пулы. Это автоматически распределит задачи по всем ядрам вашего CPU.
```python
from concurrent.futures import ProcessPoolExecutor

def square(n):
    return n * n

if __name__ == "__main__":
    numbers = [1, 2, 3, 4, 5]
    with ProcessPoolExecutor() as executor:
        results = list(executor.map(square, numbers))
    print(results) # [1, 4, 9, 16, 25]

```

# Управление потоками на Python
*Поток (Tread)* -  это поток (подпроцесс/нить/задача) внутри процесса, исполняющие некоторый код. Поотоки могут выполняться параллельно — без предписанного порядка во времени. Это позволяет: 
- Одновременно выполнять разные задачи. Например, один поток может обрабатывать данные, в то время как другой выводит информацию на экран.
- Ускорять вычисления — особенно эффективно на многоядерных системах, где каждая задача может быть распределена на отдельное ядро.
- Лучше использовать системные ресурсы — если один поток приостанавливается из-за ожидания завершения блокирующей операции (например, ввода-вывода), другие потоки могут продолжать работу, не простаивая.

    Однако из-за Global Interpreter Lock (GIL) потоки в стандартном Python (CPython) не могут выполняться на разных ядрах процессора одновременно — они работают по очереди, быстро переключаясь.

1. Импортировать модуль threading
` import threading`
2. Определить задачу (функцию)
```python
import time

def task(name, delay):
    print(f"Поток {name} начал работу")
    time.sleep(delay)  # имитация долгой задачи
    print(f"Поток {name} завершил работу")
```
3. Создать объект потока
```python
thread1 = threading.Thread(target=task, args=("A", 2))
thread2 = threading.Thread(target=task, args=("B", 4))
```
4. Запуститьть пток
```python
thread1.start()
thread2.start()
```

Пример программы :

```python
import threading
import time

def task(name, delay):
    print(f"Поток {name} начал работу")
    time.sleep(delay)  # имитация долгой задачи
    print(f"Поток {name} завершил работу")

thread1 = threading.Thread(target=task, args=("A", 2))
thread2 = threading.Thread(target=task, args=("B", 4))

thread1.start() 
thread2.start()

print("Работаем")

for i in range(10):
    time.sleep(1)
    print(i)

thread1.join()
thread2.join()
print("Все задачи выполнены")
```

- threading.current_thread(): Возвращает объект текущего потока.

# ThreadPoolExecutor (Пул потоков)
Если потоков нужно много, лучше использовать пул потоков из модуля concurrent.futures. Он сам управляет созданием и переиспользованием потоков.
```python
from concurrent.futures import ThreadPoolExecutor

with ThreadPoolExecutor(max_workers=3) as executor:
    # Запускает функцию с разными аргументами
    results = executor.map(task, ["A", "B", "C"], [1, 2, 3]) #функция из предыдущ. примера

```
# Lock (Простая блокировка)
Самый популярный инструмент. Пока один поток держит «ключ» (lock.acquire()), остальные стоят в очереди. Пример с эстафетной палочкой, у кого она в руке тот и бежит, остальные ждут пока её им передадут.

```python
import threading

lock = threading.Lock()
counter = 0

def increment():
    global counter
    with lock:  # Автоматически захватывает и освобождает замок
        counter += 1

# Без Lock счетчик мог бы посчитаться неверно из-за одновременной записи

```





# Семафор
Семафор (threading.Semaphore) — это примитив синхронизации, который работает как счетчик. В отличие от обычного Lock, который разрешает доступ только одному потоку, семафор позволяет ограничить количество потоков, одновременно работающих с ресурсом. Пример с очередью в какое-нибуд популярное кафе(булочной): одновременно в нем могут находится n человек, остальные ждут своей очереди снаружи.

### Как это работает
При создании семафора задается начальное значение счетчика (например, 3).
acquire(): Уменьшает счетчик на 1. Если счетчик равен 0, поток блокируется и ждет, пока кто-то другой не освободит место.
release(): Увеличивает счетчик на 1, позволяя одному из ожидающих потоков продолжить работу.

### Пример 1
```python
max_connections = threading.Semaphore(3)

def access_db():
    with max_connections:
        # Здесь одновременно может находиться не более 3-х потоков
        pass

```
### Пример 2

```python
import threading
import time
import random

# Разрешаем только 2 потока одновременно
pool = threading.Semaphore(2)

def worker(name):
    print(f"Поток {name} ждет доступа...")
    # Можно использовать менеджер контекста with, он сам вызовет acquire и release
    with pool:
        print(f"Поток {name} получил доступ и начал работу")
        time.sleep(random.uniform(1, 3))  # Имитация работы
        print(f"Поток {name} закончил и освободил место")

# Запускаем 5 потоков
for i in range(5):
    threading.Thread(target=worker, args=(f"#{i}",)).start()

```
# threading.Event
Класс threading.Event — это простейший механизм связи: один поток подает сигнал (флаг), а другие его ждут. У него всего два состояния: установлен (True) или сброшен (False).
Основные методы:
set(): устанавливает флаг в True. Все ожидающие потоки тут же просыпаются.
wait(): блокирует поток, пока флаг не станет True.
clear(): сбрасывает флаг в False.

В этом сценарии один поток (Producer) подготавливает данные и «мигает» событием, а другой (Consumer) реагирует на это.

```python 
import threading
import time

# Создаем объект события
message_ready = threading.Event()
# Общая переменная для данных (сообщение)
shared_data = None

def sender():
    global shared_data
    print("[Sender] Готовлю сообщение...")
    time.sleep(2)  # Имитация долгой подготовки
    shared_data = "Привет из потока-отправителя!"
    
    print("[Sender] Сообщение готово. Подаю сигнал!")
    message_ready.set() # Устанавливаем флаг, разблокируя ожидающих

def receiver():
    print("[Receiver] Жду сигнал о сообщении...")
    message_ready.wait() # Ждем, пока флаг не станет True
    
    print(f"[Receiver] Сигнал получен! Читаю данные: {shared_data}")
    message_ready.clear() # Сбрасываем флаг для следующего цикла (если нужно)

# Создаем и запускаем потоки
t1 = threading.Thread(target=sender)
t2 = threading.Thread(target=receiver)

t2.start()
t1.start()

t1.join()
t2.join()

```
